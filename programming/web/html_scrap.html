1 HTML Email 编写指南

Source

带格式的邮件，其实就是一张网页，正式名称叫做 HTML Email。它能否正常显示，完全取决于邮件客户端。大多数的邮件客户端会过滤 HTML 设置，让邮件面目全非。 编写 HTML Email 的窍门，就是使用 15 年前的网页制作方法。

1.1 DOCTYPE

目前兼容性最好的 DOCTYPE 是 XHTML 1.0 Strict=，事实上 Gmail 和 Hotmail 会删掉原本的 =DOCTYPE ，换上这个。使用这个 DOCTYPE 也就意味着不能使用 HTML5 语法。

1.2 MIME

Content-Type: text/plain;

Content-Type: Multipart/Alternative;

1.3 布局

网页的布局必须使用 <table> 。 

<body style="margin: 0; padding: 0;">
    <table border="1" cellpadding="0" cellspacing="0" width="100%">
    </table>
</body>

border 属性等于 1 是为了方便开发，正式发布的时候再把这个属性设为 0。 

<table align="center" border="1" cellpadding="0" cellspacing="0" width="600" style="border-collapse: collapse;">
    <tr></tr>
    <tr></tr>
    <tr></tr>
</table>

宽度定为 600 像素，防止超过客户端的显示宽度。

1.4 图片

图片是唯一可以引用的外部资源，其他的外部资源，比如 CSS 文件、字体文件、视频文件等，一概不能引用。 

img { outline: none; text-decoration: none; -ms-interpolation-mode: bicubic; }
a img { border: none; }

<img border="0" style="display:block;"/>

不少客户端默认不显示图片，所以要确保即使没有图片，主要内容也能被阅读。

1.5 行内样式

所有的 CSS 规则最好都采用行内样式，因为放置在网页头部的样式，很可能会被客户端删除。客户端对 CSS 规则的支持情况：CSS Support Guide for Email Clients 

<p style="margin: 1em 0;">
<p style="margin-top: 1em; margin-bottom: 1em; margin-left: 0; margin-right: 0;">

1.6 W3C 校验和测试工具

要保证最终的代码，能够通过 W3C Markup Validation Service 的校验，因为某些客户端会把不合格属性剥离。还要使用测试工具，查看在不同客户端的显示结果：

  • MailChimp Inbox Preview
  • Litmus
  • Email on Acid

发送工具可以考虑使用 MailChimp 和 Campaign Monitor。
