数据库 Database

1.1 Collection

  • 一组相关的 document，共享索引；

1.2 Document

  • 键值对，BSON 格式；

1.3 关联

Reference

  • 类似 RDBMS 的外链；
  • 用于复杂的多对多关系，或者复杂层级数据；
  • 更灵活，可以减少冗余数据

Embedded document

  • 用于一对一、一对多的「包含」关系；
  • 「读」性能更好，可以用一次操作读取、更新相关数据；

4.3 find()

db.<collection>.find(
    <condition>,
    <projection>
).<modifier>()

<condition>

  类比 WHERE
{<key>:<value>} <key> = <value>
{<key>:{$lt:<value>}} <key> < <value>
{<key>:{$lte:<value>}} <key> <= <value>
{<key>:{$gt:<value>}} <key> > <value>
{<key>:{$ge:<value>}} <key> >= <value>
{<key>:{$ne:<value>}} <key> != <value>
{<cond1>, <cond2>} <cond1> AND <cond2>
{$or: [{<cond1>}, {<cond2>}]} <cond1> OR <cond2>

<projection>

  • 指定查询返回的列，相当于 SELECT <key>, ... ；
    • 查询默认返回所有列；
    • {<key1>: 1, <key2>: 0} 表示包含 <key1> 列，排除 <key2> 列；
    • _id 除非指定排除，否则默认始终包含；

<modifier>

limit()  
sort()  
pretty() 输出格式化的查询结果

4.4 记录

db.<collection>.

insert(<document>)  
update(<cond>, {$set: {<key>:<value>}}) 更新
update(<cond>, {$set: {<key>:<value>}}, {multi:true}) 更新多条
save({_id:ObjectId(...), <key>:<value>, ...}) 替换现有记录
remove(<cond>) 移除
remove(<cond>, <max>) 最多移除几条

<document> 

{
    _id: ObjectId(...),
    <key>: <value>,
    ...
}

5 命令行

服务 

sudo service mongod start
sudo service mongod stop
sudo service mongod restart

sudo mongod --config=/etc/mongod.conf

连接 

mongo

导出 

mongoexport <options>
    --host <>
    --port <>
    --username <>
    --password <>

    --db <>
    --collection <>
    --query <>

    --out <file>

## eg.
mongoexport --db vdict --collection word --out word.json

导入 

mongoimport <options>
    --db <>
    --collection <>

    --file <>

## eg.
mongoimport --db vdict --collection word --file word.json

6 安装

## 导入公共 GPG key
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 7F0CEB10

## 创建列表文件
echo "deb http://repo.mongodb.org/apt/ubuntu "$(lsb_release -sc)"/mongodb-org/3.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-3.0.list

## 重新加载本地包数据库
sudo apt-get update

## 安装 MongoDB 包（最新）
sudo apt-get install -y mongodb-org
